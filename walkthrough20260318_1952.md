## Etapa 3 — Limpieza de Código Muerto y Trazabilidad

En esta iteración se abordó deuda técnica acumulada (código inalcanzable y basura visual), aplicando refactorizaciones conservadoras para resolver _Notices_ y prevenir fugas de cursores PDO.

### 1. Fugas de cursores PDO resueltas (`configuraciones/modelo.php`)

Se modificaron **11 métodos** que presentaban el antipatrón de llamar a `return` inmediatamente antes de `closeCursor()`. Esto dejaba cursores inalcanzables que no se liberaban, y generaba potenciales _Undefined Array Key_ si el `fetch` no arrojaba resultados.

**Patrón Aplicado:**
```php
// ANTES
$filas = $consulta->fetch(PDO::FETCH_ASSOC);
return $this->nombre_archivo = $filas['NOMBRE_ARCHIVO'];
$consulta->closeCursor(); // Inalcanzable

// DESPUÉS
$filas = $consulta->fetch(PDO::FETCH_ASSOC);
$resultado = $filas['NOMBRE_ARCHIVO'] ?? null; // Null Coalescing
$consulta->closeCursor();                      // Liberación segura
return $this->nombre_archivo = $resultado;
```
**Métodos Refactorizados:** `leoArchivosBd`, `devuelvoValorTalonSelec`, `rutaXmlConfigurada`, `facB`, `facEcommerce`, `facExpo`, `selectRxnParametrosApiTiendas`, `selectRxnParametrosRutaLocal`, `selectRxnParametrosApiLocal`, `selectRxnParametrosIdEmpresa`, `leoParametroBd`.

### 2. Remoción Masiva de Comentarios de Debug (`csv/modelo.php`)

El núcleo operacional `csv/modelo.php` tenía incrustadas alrededor de **35 líneas comentadas de debug** (e.g. `//echo '<br>Estoy entrando a SNC';` o `//print_r($info);`). Estas instrucciones se removieron mediante regex automática para no afectar funcionalidad, limpiando la legibilidad.

### Resumen Validación Etapa 3
Todos los cambios fueron estructurales limpios. No se tocó lógica impositiva, acumuladores SQL intermedios (`art_total_*`) ni la función duplicada `generoJsonCompleto`. El sistema conserva compatibilidad estricta.
